## What we did

### Session A — Typography & Event Cards
Overhauled the typography system through a multi-step font exploration — tried Extenda, Big Shoulders Display, and Passion One before landing on Bungee (display) + DM Sans (body) as the final pairing. Redesigned the six event cards from 6 narrow columns into a 3×2 grid at 9:16 aspect ratio. Added a hover reveal effect where the overlay and text fade out, exposing the full background photo.

### Session B — Cup Pop-Out Animation, Asset Compression & Deployment
Built a scroll-driven arm/cup animation on the What's On section. Compressed all photos (~280MB → ~32MB) and videos (~183MB → ~36MB) using Node.js + sharp + ffmpeg-static. Initialized git repo, pushed to GitHub, and deployed live on Vercel.

## Decisions made

### Typography
- **Bungee** (Google Fonts) for all display/headings: modeled after venue marquee signage, chunky without being wide.
- **DM Sans** (locally hosted variable font) for all body text and labels.
- **Extenda rejected**: 80-Peta variant too wide, 40-Hecto better but still not right.
- **Big Shoulders Display rejected**: too thin/tall for Harper's energy.
- **Passion One rejected as sole font**: unreadable at small label sizes.

### Event Cards
- **3×2 grid at 9:16**: previous 6-column layout was too narrow. Each card has its own color identity pulled from matching promo flyer photos.
- **Hover reveal**: overlay + text fade to opacity 0, background photo zooms to scale(1.07). No JS needed.
- **Card size**: reduced to 81% max-width of container to feel slightly tighter.

### Cup Pop-Out Animation
- Image used: `Photos/Adobe Express - file (1).png` (white background cutout of hand holding Harper's blue drink)
- Blend mode: `mix-blend-mode: multiply` — makes the white background invisible against the cream section
- `filter: drop-shadow(6px 12px 20px rgba(0,0,0,0.35))` — shadow follows the cutout shape
- Positioned: `left: -60px`, `bottom: -80px`, `width: 550px` — arm extends off left edge, cup visible ~mid-left
- Animation: **scroll-driven** via `window.addEventListener('scroll')` — arm position tied directly to scroll progress through the section. NOT a CSS class toggle.
- Timing: hidden 0–55% of scroll journey, ramps in 55–75%, holds 75–85%, retracts 85–100%
- **Latch behavior**: once arm is fully extended (progress ≥ 0.75), it stays out as user continues scrolling into the next section. Only retracts when scrolling back up past the 55% threshold.
- CSS transition: `0.16s ease-out` (smooth scroll tracking)
- Hidden on screens ≤ 900px
- `position: relative` on `.thisweek` (no overflow hidden — body already has `overflow-x: hidden`)

### Patio Section
- Reduced top padding from `clamp(80px, 10vw, 140px)` to `clamp(40px, 5vw, 72px)` to tighten the gap at top of section.

### Session C — Performance Pass (scroll lag + image pop-in fix)

## What we did

Fixed 6 live performance issues causing scroll lag and images popping in late on the deployed Vercel site. Rewrote main.js to consolidate all scroll handlers into a single RAF-throttled function, added an image pre-loader observer with 600px lookahead, added a video visibility observer for the 4 promo clips, added `will-change` hints to parallax targets, created vercel.json with 1-year cache headers, and added `<link rel="preload">` resource hints for the hero poster and logo in index.html.

## Decisions made

- **6 scroll handlers → 1 RAF loop**: Nav, scroll cue, patio parallax, gameday parallax, brewery parallax, and drink arm now share a single `requestAnimationFrame` handler — eliminates per-scroll layout thrash.
- **Image pre-loader observer (600px lookahead)**: Fires before sections enter view, upgrades `loading="lazy"` → `loading="eager"` so images are decoded before reveal animations run. Fixes the "image pops in after fade" effect.
- **Video visibility observer**: The 4 `.video-item video` elements now only play when 20% visible, pause when scrolled away.
- **`will-change: transform`** on `.gameday-img`, `.brewery-img`, `.patio-photo img` — GPU-promotes parallax targets before scroll events.
- **Reveal animation tuned**: `translateY` 40px → 28px, duration 0.75s → 0.6s, delays 0.12/0.24/0.36 → 0.10/0.20/0.30.
- **vercel.json created**: Photos/Videos/Font folders get `max-age=31536000, immutable`. CSS/JS get 1-day with stale-while-revalidate.
- **Hero resource hints**: `<link rel="preload">` for hero poster + logo. `fetchpriority="high"` on hero logo img. `decoding="async"` on all lazy images.

## Files changed
- `js/main.js` — full rewrite of scroll handling + new observers
- `css/styles.css` — will-change, reveal animation tuning
- `index.html` — preload hints, fetchpriority, decoding=async
- `vercel.json` — created (cache headers for all static assets)
