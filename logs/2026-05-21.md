## What we did
Migrated the site from plain HTML/CSS/JS to Astro (static site framework) and connected Sanity as a headless CMS. The full migration happened on a separate git branch (`astro-migration`), was confirmed working, then merged to master and deployed to Vercel. Sanity is now live and the "This Week" event cards are driven by Sanity content. We also built a booking form for the Private Events section (Formspree, pending real form ID) and fixed mobile ordering so the form appears before the quote on small screens.

## Decisions made
- **Astro chosen over Next.js**: Lowest migration friction from plain HTML. CSS/JS untouched. Same output.
- **Sanity for CMS**: Harper's staff will edit weekly specials and contact info via harpers-brewpub.sanity.studio
- **Static output with build-time fetching**: Sanity content is baked in at build time (not live-fetched). Changing content in Sanity requires a Vercel redeploy to go live.
- **Fallback data**: If Sanity is unreachable or empty, the site falls back to hardcoded content automatically. Nothing breaks.
- **Account ownership**: Everything currently lives on Adam's accounts (GitHub, Vercel, Sanity). Transfer to Harper's at handoff — see Transfer section below.
- **Branch strategy**: `astro-migration` branch was used for all migration work. `master` held the working plain HTML site until confirmed working.
- **Private Events form**: Built with Formspree. `YOUR_FORM_ID` placeholder still in place — needs real Formspree account (see HANDOFF.md).

## Tech stack (current)
- **Framework**: Astro 5 (static output)
- **CMS**: Sanity v3 — project ID `h0rc5ul6`, dataset `production`
- **Sanity Studio**: https://harpers-brewpub.sanity.studio
